##  1️⃣ 时间维度指标（Time-based Features）

### 1.1 核心指标

| 指标名称 | 计算方法 | 正常范围 | 可疑模式 | 权重 |
|---------|---------|---------|---------|------|
| `mean_delta_ms` | 相邻订单时间间隔均值 | 5000-60000 | <1000（高频）| ⭐⭐⭐ |
| `std_delta_ms` | 时间间隔标准差 | >3000 | <500（极规律）| ⭐⭐⭐ |
| `cv_delta` | 变异系数 = std/mean | 0.5-2.0 | <0.15（机器人核心特征）| ⭐⭐⭐⭐⭐ |
| `min_delta_ms` | 最小时间间隔 | >300 | <100（超高频）| ⭐⭐⭐⭐ |

**变异系数（CV）阈值参考：**
- `cv < 0.1`：**极高可疑** - 固定频率机器人
- `0.1 ≤ cv < 0.2`：**高度可疑** - 策略化交易
- `0.2 ≤ cv < 0.4`：**中度可疑** - 半自动交易
- `cv ≥ 0.4`：**正常** - 人工交易

熵计算示例：
```java
double computeEntropy(List<BigDecimal> sizes) {
    Map<BigDecimal, Integer> freq = new HashMap<>();
    for (BigDecimal s : sizes) {
        freq.merge(s, 1, Integer::sum);
    }
    double entropy = 0.0;
    int total = sizes.size();
    for (int count : freq.values()) {
        double p = count / (double) total;
        entropy -= p * Math.log(p) / Math.log(2);
    }
    return entropy;
}
```

6.1 群体同步指标
时间桶定义：
将时间分为固定大小桶（如500ms），同一桶内相同Symbol+Side的订单视为"同步"。

| 指标名称                   | 说明                         | 检测阈值                  |
|----------------------------|------------------------------|---------------------------|
| sync_events_1min           | 最近 1 分钟参与同步事件次数  | > 10                      |
| sync_strength              | 同步事件数 / 总订单数        | > 0.5（高度同步）         |
| max_sync_group_size        | 参与过的最大同步组规模       | > 10 个账户               |
| account_pair_sync_count    | 与特定账户共现次数           | > 20                      |

同步检测算法：
```python
def detect_sync_events(orders, bucket_ms=500):
    buckets = {}
    for order in orders:
        bucket = (order.ts // bucket_ms) * bucket_ms
        key = f"{order.symbol}|{order.side}"
        buckets.setdefault(bucket, {}).setdefault(key, []).append(order.account_id)
    
    sync_events = []
    for bucket, groups in buckets.items():
        for key, accounts in groups.items():
            if len(accounts) >= 3:  # 至少3个账户同步
                sync_events.append({
                    'bucket_ms': bucket,
                    'symbol_side': key,
                    'accounts': accounts,
                    'count': len(accounts)
                })
    return sync_events
```

```java
// 特征向量：[cvDelta, cancelRatio, avgTPSL, orderSizeEntropy, ...]
double cosineSimilarity(double[] vecA, double[] vecB) {
    double dotProduct = 0.0, normA = 0.0, normB = 0.0;
    for (int i = 0; i < vecA.length; i++) {
        dotProduct += vecA[i] * vecB[i];
        normA += vecA[i] * vecA[i];
        normB += vecB[i] * vecB[i];
    }
    return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}
```

| 80-100   | L4       | 极高风险 | 立即禁用，人工复核     |
| 60-79    | L3       | 高风险   | 限制交易，加强监控     |
| 40-59    | L2       | 中风险   | 增加延迟，降低限额     |
| 20-39    | L1       | 低风险   | 轻度监控               |
| 0-19     | L0       | 正常     | 无限制                 |

# 📈 使用示例
示例1：识别固定策略机器人
```sql
-- 从特征表查询高度可疑账户
SELECT 
    account_id,
    cv_delta,
    identical_tpsl_ratio,
    order_size_entropy,
    bot_likelihood_score
FROM account_features_enhanced
WHERE 
    cv_delta < 0.15
    AND identical_tpsl_ratio > 0.8
    AND order_size_entropy < 1.0
    AND bot_likelihood_score > 60
ORDER BY bot_likelihood_score DESC
LIMIT 50;

```
示例2：发现群体同步操控
```python
-- 查找同步事件中的账户群组
SELECT 
    bucket_ms,
    symbol_side,
    account_count,
    LEFT(account_ids, 100) as sample_accounts
FROM sync_events
WHERE account_count >= 5
ORDER BY account_count DESC, bucket_ms DESC
LIMIT 100;

```
示例3：延迟套利检测
```python
-- 高 quote_age + 正 markout = 陈旧报价套利
SELECT 
    a.account_id,
    a.avg_quote_age_ms,
    a.stale_quote_ratio,
    m.avg_markout_500ms,
    m.pos_ratio_500ms
FROM account_features_enhanced a
JOIN markout_agg_by_account m ON a.account_id = m.account_id
WHERE 
    a.avg_quote_age_ms > 200
    AND a.stale_quote_ratio > 0.2
    AND m.avg_markout_500ms > 0.00005
    AND m.pos_ratio_500ms > 0.65
ORDER BY m.avg_markout_500ms DESC;

```
# 🔧 技术实现备注
计算复杂度
| 指标类型         | 时间复杂度            | 空间复杂度                | 实时性               |
|------------------|-----------------------|---------------------------|----------------------|
| 时间间隔统计     | O(1) 均摊             | O(N) 环形缓冲区           | 实时                 |
| 订单状态追踪     | O(1) 哈希表           | O(M) pending 订单数       | 实时                 |
| 同步检测         | O(N) 时间桶遍历       | O(N × K) 桶 × 账户        | 准实时（1–5s 延迟）  |
| 熵计算           | O(N) 遍历             | O(U) 唯一值数             | 离线                 |
| 相似度聚类       | O(N²) 两两比较        | O(N) 特征向量             | 离线                 |

性能优化建议
- **采样策略**：对于低优先级指标（如熵、相关性），采用采样计算而非全量遍历  
- **异步计算**：将复杂指标（如聚类、网络分析）放入异步线程或批处理任务  
- **增量更新**：使用 RollingStats 等增量统计方法，避免重复扫描历史数据  
- **LRU 淘汰**：对不活跃账户定期清理状态数据，控制内存占用
