# 架构说明

## 总体定位

`asu-risk-manage` 采用按能力模块拆分的方式组织系统，而不是把所有逻辑堆进单一应用。

当前主链路为：

`offline / batch / online -> feature / rule-engine / graph -> risk -> case -> ai`

其中：
- `fraud-core` 是横向共享层
- `fraud-batch` 是定期批处理编排入口
- `fraud-online` 是查询与重算入口
- `fraud-risk` 是统一风险评分主落点
- `fraud-case` 与 `fraud-ai` 负责结果解释和调查闭环

当前基础运行版本：
- JDK 25
- Spring Boot 3.5.7

## 关键模块边界

### `fraud-feature`

职责：
- 特征定义
- 特征生成
- 特征快照与历史存储
- 对外特征读取视图

不负责：
- 最终风险总分
- 案件生成
- 离线 replay 分析

### `fraud-rule-engine`

职责：
- 规则定义与生效配置装载
- 规则参数解析
- 规则执行
- 规则命中聚合与日志记录

不负责：
- 最终风险总分
- 特征生成
- 案件组装

当前定位：
- 独立模块：有必要
- 独立业务主程序：没有必要

### `fraud-risk`

职责：
- 统一风险评分编排
- 汇总规则、图、ML、行为等信号
- 风险等级判定
- 原因码生成
- 风险结果持久化

当前内部边界：
- [RiskEngineFacade](fraud-risk/src/main/java/me/asu/ta/risk/service/RiskEngineFacade.java)
  - 跨模块稳定入口
- [RiskEvaluationService](fraud-risk/src/main/java/me/asu/ta/risk/service/RiskEvaluationService.java)
  - 评分编排层
- [GraphRiskSignalResolver](fraud-risk/src/main/java/me/asu/ta/risk/service/GraphRiskSignalResolver.java)
  - 图信号统一解析
- [BehaviorScoreCalculator](fraud-risk/src/main/java/me/asu/ta/risk/scoring/BehaviorScoreCalculator.java)
  - 行为信号计算
- [RiskReasonGenerator](fraud-risk/src/main/java/me/asu/ta/risk/reason/RiskReasonGenerator.java)
  - 原因码候选合并
- [RiskScoreResultFactory](fraud-risk/src/main/java/me/asu/ta/risk/service/RiskScoreResultFactory.java)
  - 最终结果装配

当前定位：
- 核心是 `lib`
- 保留轻量 `app` 外壳
- 不演化成总控程序

### `fraud-offline`

职责：
- 文件输入
- replay / markout / quote-age
- baseline / anomaly / bot indicators
- 行为向量 / 行为分群 / 相似度边
- 可选接入 `feature / risk / case`

行为分析能力：
- [BehaviorFeatureAnalysisService](fraud-offline/src/main/java/me/asu/ta/offline/analysis/BehaviorFeatureAnalysisService.java)
- [BehaviorClusterAnalysisService](fraud-offline/src/main/java/me/asu/ta/offline/analysis/BehaviorClusterAnalysisService.java)
- [AccountSimilarityAnalysisService](fraud-offline/src/main/java/me/asu/ta/offline/analysis/AccountSimilarityAnalysisService.java)

这些结果的语义是：
- `behavior cluster`
- `behavior similarity edge`

它们不是实体关系图。

### `fraud-batch`

职责：
- 定期任务编排
- 串联 `feature -> risk -> case`
- 统一批处理入口

第一阶段提供：
- `FeatureRefreshBatchJob`
- `RiskEvaluationBatchJob`
- `CaseGenerationBatchJob`
- `PIPELINE` 编排模式

约束：
- 不实现评分逻辑
- 不直接承载离线 replay 分析
- 复用已有 `FeatureGenerationJobRunner`、`RiskEvaluationJobRunner`、`CaseGenerationJobRunner`

### `fraud-online`

职责：
- 查询 API
- 单账号手动重算
- 事件驱动风险重算
- 风险 job 查询

当前不负责：
- 完整实时撮合决策
- 规则内核实现
- 风险评分实现

查询读模型当前按账号维度装配：
- 最新风险结果
- 风险历史
- 最新特征与历史
- 最近规则命中
- 最近案件摘要

### `fraud-case`

职责：
- 调查案件对象
- 风险摘要、规则摘要、图摘要
- 时间线与推荐动作
- 接收统一风险结果和上下文信号做解释增强

### `fraud-ai`

职责：
- 读取案件对象
- 组织 prompt
- 调用模型
- 解析和持久化调查报告

## 调用关系

### 1. 定期批量主链路

1. 调度器触发 `fraud-batch`
2. `fraud-batch` 刷新 `AccountFeatureSnapshot`
3. `fraud-batch` 调用 `RiskEngineFacade`
4. `fraud-risk` 调用 `RuleEngineFacade`
5. 生成并持久化 `RiskScoreResult`
6. 对高风险账户触发 `fraud-case`

### 2. 事件驱动重算链路

1. 事件进入 `fraud-online`
2. 读取最新 `AccountFeatureSnapshot`
3. 按事件类型补充 `contextSignals`
4. 调用 `RiskEngineFacade`
5. 生成并返回最新风险结果

### 3. 查询链路

1. 查询请求进入 `fraud-online`
2. 优先读取最新 `RiskScoreResult`
3. 补充规则命中、特征快照、案件摘要
4. 返回账号风险总览或历史结果

### 4. 离线并入主链路

1. `fraud-offline` 完成 replay 和行为分析
2. 构造 `RiskEvaluationRequest`
3. 调用 `RiskEngineFacade`
4. 风险结果进入 `risk / case / ai`

## Facade 使用原则

当前项目不追求“到处使用 facade”，只保留少量有明确价值的跨模块入口：

- 保留：
  - `RiskEngineFacade`
  - `RuleEngineFacade`
- 避免：
  - 同模块内部的薄转发 facade
  - 仅为了模式而新增 facade

判断标准：
- 跨模块稳定入口：适合 facade
- 同模块内部包装：不必 facade

## 查询主来源约定

`fraud-online` 默认查落库结果，不隐式重算。

推荐查询优先级：
1. 最新风险：`RiskScoreResult`
2. 风险历史：`RiskScoreResult` 历史记录
3. 最新特征：`AccountFeatureSnapshot`
4. 特征历史：`AccountFeatureHistory`
5. 规则命中：`rule_hit_log`
6. 案件摘要：`fraud-case` 查询服务
