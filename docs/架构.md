使用建议
有三个实践建议最重要：
第一，不要让模型一次性输出全项目源码。
即使是强模型，一次全出也很容易出现模块边界混乱、重复类、依赖膨胀和命名漂移。
第二，每个 Prompt 都要加当前项目上下文。
比如你已经生成了 fraud-core，再生成 fraud-ingestion 时，把 fraud-core 的关键类定义附上，模型会稳定很多。
第三，在每一轮生成后做一轮“依赖减法审查”。
大模型很喜欢“顺手加库”，但你的目标是生产级稳健系统，不是 demo。
